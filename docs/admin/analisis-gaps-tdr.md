# Análisis de GAPS - TDR vs Planificación Proyecto

**Documento**: Comparación entre requisitos TDR FICEM-ASOCEM-PRODUCE vs Planificación Actual Proyecto LATAM-3C
**Fecha**: 2025-11-05
**Total GAPS**: 23

---

## ÍTEM 1: PROTOCOLO CAPTURE/CÁLCULOS/ANÁLISIS/CONSISTENCIA PRODUCE

1. **Ausencia de protocolo de análisis de consistencia específico para PRODUCE**
   - TDR requiere: Análisis de consistencia según criterios comparativos/benchmarking
   - Planificación: Validación multinivel no cubre análisis específico PRODUCE

2. **Ausencia de documento estructura y flujos de información para PRODUCE**
   - TDR requiere: Definición de estructura y flujos de información requeridos por PRODUCE
   - Planificación: Asume CSI Protocol 3.1 sin documento específico PRODUCE

3. **Ausencia de herramienta de solicitud anual de datos a empresas**
   - TDR requiere: Herramienta para solicitar datos a empresas (separada de cálculos)
   - Planificación: Solo generador Excel para FICEM, sin herramienta de solicitud PRODUCE

4. **Falta definición del ciclo anual PRODUCE explícito**
   - TDR requiere: Ciclo anual de recepción, validación, reporte
   - Planificación: No especifica ciclo PRODUCE

## ÍTEM 2: SISTEMA MRV HR PERÚ

5. **Ausencia de protocolo específico "HR Perú" diferenciado**
   - TDR requiere: Protocolo propio para HR Perú
   - Planificación: Sistema MRV genérico sin diferenciación HR Perú

6. **Ausencia de planilla estandarizada Excel oficial**
   - TDR requiere: Planilla Excel de medición y reporte de emisiones
   - Planificación: Dashboard Streamlit, pero sin planilla Excel formal

7. **Falta mecanismo de coordinación ASOCEM-FICEM-PRODUCE documentado**
   - TDR requiere: Coordinación con ASOCEM y FICEM para integración de resultados
   - Planificación: No define proceso multi-stakeholder para validación conjunta

8. **Falta desglose de Fase 1 (Dic 2025) vs Fase 2 (Jun 2026)**
   - TDR requiere: Implementación Dic 2025 + ajuste/consolidación Jun 2026
   - Planificación: Timeline único sin diferenciación entre fases

9. **Ausencia de manual de procedimientos de confidencialidad**
   - TDR requiere: Procedimientos de confidencialidad y resguardo de datos
   - Planificación: Solo menciona pseudonimización sin documento operativo

## ÍTEM 3: CALCULADORA PERÚ 4C

10. **Falta metodología para curvas CO₂ vs Resistencia país-específico**
    - TDR requiere: Curvas promedio país de CO₂ vs Resistencia (Perú)
    - Planificación: Curvas FICEM genéricas/regionales, no país-específico

11. **Ausencia de sistema operativo de ranking de productos bajos en carbono**
    - TDR requiere: Ranking de productos bajos en carbono
    - Planificación: Solo benchmarking anónimo, sin ranking oficial

12. **Falta diferenciación de protocolos por tipo de producto**
    - TDR requiere: Protocolos separados clínker/cemento/concreto/CO₂
    - Planificación: Protocolo genérico para todos los productos

13. **Ausencia de plan de capacitación estructurado**
    - TDR requiere: Validación y capacitación para uso de herramienta
    - Planificación: Solo "validación con empresas piloto" sin programa formal

14. **Falta documento formal "Protocolo Calculadora Perú 4C" como entregable**
    - TDR requiere: Entregable separado protocolo + herramienta
    - Planificación: No diferencia documento de protocolo vs herramienta

## CRÍTICOS IDENTIFICADOS

1. **Falta diferenciación entre protocolos PRODUCE vs HR Perú vs Calculadora 4C**
   - Actualmente planificación trata como genéricos
   - TDR requiere 3 protocolos distintos con entregables separados

2. **Falta coordinación multi-stakeholder documentada**
   - TDR requiere roles explícitos: FICEM (técnica regional) + ASOCEM (nacional) + PRODUCE (validación)
   - Planificación no define governance

3. **Desalineación cronológica**
   - Proyecto termina Mar 2026, TDR requiere Jun 2026
   - Riesgo: Fases HR Perú y Calculadora 4C pueden llegar incompletas

4. **Falta diferenciación país-específico en curvas y benchmarking**
   - TDR requiere curvas Perú específicas
   - Planificación usa metodología FICEM genérica regional
